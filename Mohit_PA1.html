We have implemented neural network with one hidden layer using the provided Logistic Regression template.
We will implement a simple 2 layer neural network with following specification.

  1. Layer 1 :
    Input = Number of features
    Output = Number of nodes in hidden layer. i.e 20 Activation function = Sigmoid

  2. Layer 2 :
    Input = Number of nodes in hidden layer
    Output = Number of dimensions in the output = number of classes in y
    Activation function = None
    The ouput will be passed through softmax function to intepret the output as probabilities

#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""

# sigmoid function
def sig(h):
    sig_h = np.exp(h)/(1+np.exp(h))
    return sig_h;

#softmax function
def softmax(z):
    exp_out = np.exp(z)
    softmax_scores = exp_out / np.sum(exp_out, axis=1, keepdims=True)
    return softmax_scores

def softmax_y(softmax_scores):
    return np.argmax(softmax_scores, axis=1)

class one_layer_nn():

    #weights Initialization
    def __init__(self,in_dim, nodes, out_dim, alpha=0.5 ):
        self.alpha=alpha
        self.w_hidden = np.random.randn(in_dim, nodes) / np.sqrt(in_dim)
        self.b_hidden = np.zeros((1, nodes))
        self.w_out = np.random.randn(nodes, out_dim)/np.sqrt(nodes)
        self.b_out = np.zeros((1, out_dim))

    #Calculates the activations in different layer
    def forward_prop(self,X_train):
        #input for hidden layer h
        in_h = np.dot(X_train, self.w_hidden) + self.b_hidden
        #passing through sigmoid function
        o_h = sig(in_h)
        #input to last layer z
        in_out = np.dot(o_h, self.w_out) + self.b_out
        #passing through softmax dunction
        o_out = softmax(in_out)
        activation = {"o_h": o_h , "o_out":o_out}
        return activation


    #Calculate the gradient wrt to theta and bias terms
    def back_prop(self,activation,X_train, y_train,m):
        #dummy type
        one_hot_y = np.zeros_like(activation["o_out"])
        for i in range(y_train.shape[0]):
            one_hot_y[i, y_train[i]] = 1

        
        o_out = activation["o_out"]
        #d(Performace) / d(net_out)
        dp_net_out = o_out - one_hot_y
        beta_out = dp_net_out
        o_h = activation["o_h"]
        
        #net_out = (o_h * w_out) + b_out
        #d(performance)/d(net_out) * d(net_out)/d(w_out)
        dp_w_out = np.dot(o_h.T, beta_out)/m 
        #d(performance)/d(net_out) * d(net_out)/d(b_out)
        dp_b_out = np.sum(beta_out, axis=0, keepdims=True)/m # dp/dp_net_out  * dp_net_out/db


        #d(performance)/d(o_h) = d(performance) / d(net_out) *  d(net_out) / d(o_h)
        dp_o_h = np.dot(beta_out, self.w_out.T)
        
        #net_h = (X * w_h) + b_h
        #d(o_h)/d(net_h)
        doh_net_h = np.multiply(o_h, 1 - o_h)
        #d(performance)/d(net_h) = d(performance) / d(o_h) * d(o_h) /d(net_h) = dp_o_h * doh_net_h
        dp_net_h = np.multiply(dp_o_h, doh_net_h)
        #Calcuation for weight update in hidden layer
        beta_h = dp_net_h
        
        #d(performance)/d(net_h) * d(net_h)/d(w_h)
        dp_w_h = np.dot(X_train.T, beta_h)/m
        #d(performance)/d(net_h) * d(net_h)/d(b_h)
        dp_b_h = np.sum(beta_h, axis=0, keepdims=True)/m

        #Return updated gradients
        grad_des = {"w_out" : dp_w_out ,
                  "b_out" : dp_b_out ,
                  "w_h": dp_w_h ,
                  "b_h": dp_b_h}
        return grad_des

    def compute_cost(self,o_out,y,m):
        return  np.sum(1-(o_out == y))/m

    def update_weights (self, grad_des):
        self.b_hidden -= self.alpha*grad_des["b_h"]
        self.b_out -= self.alpha*grad_des["b_out"]
        self.w_hidden -= self.alpha*grad_des["w_h"]
        self.w_out -= self.alpha*grad_des["w_out"]

    def fit(self, X_train,y_train, num_epochs):
        train_error = []
        for i in range(num_epochs):
            activation = self.forward_prop( X_train)
            grad_des  = self.back_prop( activation,X_train, y_train,X_train.shape[0])
            self.update_weights(grad_des)
            if(i%10 == 0):
                #output of last layer(z)
                z_out = softmax_y(activation["o_out"])
                train_error += [self.compute_cost(z_out, np.array(y_train).T, X_train.shape[0])]
        return train_error

    def predict(self, X_test):
        output = self.forward_prop(X_test)
        o_out = output["o_out"]
        return softmax_y(o_out)

def k_fold(X,y,k,nn):
    kf = KFold(n_splits=k, random_state=None, shuffle=False)

         train_error = (nn.fit(X_train, y_train, num_epochs = 2500))

         test_error=(nn.compute_cost(y_pred, y_test, X.shape[0]))

    return (np.sum(test_error)/(k+1))

The cross validation procedure enables you to combine the results of five experiments. Why is this useful? ¶

If we train our network on the given dataset and test on the real data, we can end up having a model that is underfitted or overfitted. Because, we get an idea of how well our model is performing on the dataset we used to train our model, but we will be unsure about its performance on unseen dataset. Thus,
by using K Fold cross validation, we divide the data into k subset and perform the same procedure k time. In each iteration, we take one of the k subsets for validation set and other k-1 for training set. When we take the error estimation for all k iteration and get their average, we find the effectiveness of our model. As we use the most of our data for fitting, we significantly reduce bias of our network as well the the variance as we use these data in validation. Therefore, we can say that interchanging the training and testing dataset improves the effectiveness of our system.

What is overfitting and why does it occur in practice? Name and briefly explain 3 ways to reduce overfitting.¶

When a model tries to learn all the details and noises in the training data and creates a convoluted model to closely fit the data points such that it doesn’t generalize well for the unseen data, is called overfitting. It occurs because the model attempts to pick up the noises in the training data and learn them as a concept, thus, when it comes to predicting the unseen data, it performs miserably.

  1. L2 regularization: We use L2 Regularization to reduce the variance of the estimator by simplifying it. Which will increase the bias, however, our expected error decreases.

  2. Wrapper: We can use wrapper to enumerate the model with respect to the number of nodes in the neural network. Thus, we will have list of models with respective errors and we can chose the model with minimum error.

  3. Feature selection: The core issue with overfitting is that it has more number of independent parameters than the number of data points. Thus, by discarding irrelevant attributes we can avoid overfitting.

class regularized_nn(one_layer_nn):
    
    def __init__(self,input_dim, nodes, output_dim , alpha = 0.1,reg_para=0.0001):
        self.alpha = alpha
        self.reg = reg_para
        super().__init__(input_dim, nodes, output_dim,alpha )
            
    def update_weight (self, grad_des):
        #Adding derivative of regularization term
        self.b_hidden -= self.reg*self.b_hidden
        self.b_out -= self.reg*self.b_out
        self.w_hidden-= self.reg*self.w_hidden
        self.w_out-= self.reg*self.w_out
        
        self.b_hidden -= self.alpha*grad_des["b_h"] 
        self.b_out -= self.alpha*grad_des["b_out"]
        self.w_hidden -= self.alpha*grad_des["w_h"]
        self.w_out -= self.alpha*grad_des["w_out"]

One common technique used to reduce overfitting is L2 regularization. How does L2 regularization prevent overfitting? How differently does your model perform before and after implementing L2 regularization?¶

To prevent overfitting by L2 regularization, we introduce a cost term for bringing in more features with the objective function. Which means we will penalize the hypothesis complexity. Therefore, it tries to push the coefficients for many variables to zero and hence reduce cost term.

What effect does the learning rate have on how your neural network is trained?¶

We know that the learning rate decides at what pace we are going to converge. Therefore, if we take a small learning rate, we will move slowly towards the minima, however, it would take very long time to converge. Furthermore, if we take a large learning rate, we can miss the local minima and maybe this will never converge. Hence, if we chose an optimal learning rate, we would spend lesser time to train our neural network.
